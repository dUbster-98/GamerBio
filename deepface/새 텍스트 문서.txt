● 프로덕션에서는 "PC가 RPi5(LAN)로 직접 POST"하는 구조라, 로컬과 달라지는 핵심은 ① 대상 URL을 RPi5로, ② API 키 포함, ③
  RPi5에 사진 저장 경로 설정, ④ 새 코드 배포입니다. 하나씩 짚어드릴게요.

  1) PC — DeepFace를 RPi5로 향하게 실행

  프로덕션 RPi5는 LAN에서 평문 HTTP(http://192.168.0.104:5000)로 받고, /api는 API 키를 요구합니다(현재
  appsettings.Production.json에 BioMonitor:ApiKey가 설정돼 있죠). 그래서:

  python deepface/emotion_webcam.py --stream --headless `
    --capture-url http://192.168.0.104:5000/api/gallery/capture `
    --post-url    http://192.168.0.104:5000/api/emotion `
    --api-key <프로덕션_BioMonitor:ApiKey_값> `
    --no-insecure

  - --api-key는 필수입니다. 빠지면 /api/gallery/capture가 401을 돌려주고 캡처가 안 됩니다(기존 감정 송신에 쓰던 그 키
  그대로).
  - --no-insecure: 평문 http라 TLS 검증은 무의미하지만, 로컬 기본값(DEFAULT_INSECURE=True)을 끄는 의미로 붙여둡니다.
  - --headless: 화면 창 없이 돌릴 때.

  ▎ 매번 타이핑이 싫으면 파일 상단 DEFAULT_* 블록을 프로덕션 값으로 바꿔두면 됩니다(DEFAULT_CAPTURE_URL=RPi5 주소,
  ▎ DEFAULT_API_KEY=키, DEFAULT_INSECURE=False). 다만 그 파일은 git에 올라가니 키를 커밋하기 싫으면 CLI --api-key로
  ▎ 넘기는 쪽을 권합니다.

  2) RPi5 — 사진 저장 경로를 배포 폴더 밖으로 (중요)

  기본 저장 경로는 ContentRoot/gallery-store = /opt/biomonitor/gallery-store인데, 배포 때 /opt/biomonitor를 덮어쓰면
  사진이 날아갑니다. RPi5의 appsettings.Production.json에 배포 폴더 밖 경로를 지정하세요:

  {
    "Gallery": {
      "StoragePath": "/home/tjdgus/gallery-store"
    }
  }

  - 앱이 시작 시 그 폴더를 자동 생성(Directory.CreateDirectory)하고, 서비스가 tjdgus로 도니 /home/tjdgus/...는 쓰기
  가능합니다.
  - 서버 측 자동캡처(AutoCapture)는 손댈 것 없습니다 — base에서 Enabled:false라 PC 캡처와 중복되지 않습니다.
  (Production.json에 굳이 넣지 마세요.)

  3) 새 코드 배포 (기존 파이프라인 그대로)

  갤러리 + 캡처 엔드포인트 + 마이그레이션이 들어갔으니 재배포가 필요합니다:

  dotnet publish GamerBio/GamerBio.csproj -c Release -r linux-arm64 --self-contained false -o publish
  scp -r publish/* tjdgus@192.168.0.104:/opt/biomonitor/
  ssh tjdgus@192.168.0.104 "sudo systemctl restart biomonitor-api.service"

  - 앱 시작 시 MigrateAsync()가 AddGalleryPhotos 마이그레이션을 적용해 PostgreSQL에 gallery_photos 테이블을 자동
  생성합니다. 별도 작업 불필요.
  - 이번엔 InMemory가 아니라 Postgres라, 캡처된 사진 메타데이터가 영구 보존됩니다(파일은 위 StoragePath에).

  4) 동작 확인

  - RPi5 로그: ssh … "journalctl -u biomonitor-api -f" → 캡처 시 PC capture saved: surprise 9x → gallery id=..
  - PC 콘솔: [capture] sent surprise 9x% (200) → http://192.168.0.104:5000/...
  - 브라우저: https://bio-monitor.uk/gallery (Cloudflare Tunnel 경유)에서 사진 확인 — 캡처는 LAN 직통이라 터널 설정은
  건드릴 것 없고, 외부 열람만 터널을 탑니다.

  참고

  - 캡처 POST는 LAN 직통(PC→RPi5:5000)이라 Cloudflare Tunnel/도메인 설정과 무관합니다.
  - 임계값을 프로덕션에서 바꾸려면 PC 쪽 --capture-threshold(또는 DEFAULT_CAPTURE_THRESHOLD)만 조절하면 됩니다. 서버
  재배포 불필요.

  요약하면 PC 명령에 RPi5 주소 + API 키만 맞추고, RPi5엔 Gallery:StoragePath만 추가한 뒤 재배포하면 됩니다. 배포
  스크립트(publish→scp→restart)를 한 번에 실행하는 .ps1로 묶어드릴까요?
